
## Что делаем

В разделе «Цели» (`src/routes/_app.wishes.tsx`) добавляем возможность связывать цели иерархически: у цели может быть одна над-цель и несколько под-целей, глубина не ограничена. Связи видны прямо на карточке в списке, а редактируются из открытой цели в обе стороны.

## Модель данных

Расширяем `interface Goal`:

```ts
interface Goal {
  // ...existing
  parentGoalId?: string | null;  // одна над-цель
}
```

Под-цели вычисляются: `children = goals.filter(g => g.parentGoalId === goal.id)`.
Одна над-цель на цель — это дерево, но глубина не ограничена (цепочка над-целей → под-целей любой длины). Защищаемся от циклов: при выборе над-цели исключаем саму цель и всех её потомков.

## Где показываем — карточка в списке

Под заголовком цели добавляем компактную строку связей (только если они есть):

```text
[картинка]  Пробежать первый марафон
            ↑ Здоровое тело · ↓ 3 под-цели
            [прогресс-бар]
```

- `↑ Название над-цели` — маленький чип, tap → скролл/открытие над-цели.
- `↓ N под-целей` — чип со счётчиком, tap → открывает цель и разворачивает секцию «Под-цели».
- Оба чипа — 11–12px, приглушённый цвет, чтобы не спорить с заголовком.
- Если связей нет — строка не рендерится, карточка выглядит как сейчас.

## Где редактируем — внутри цели

В экране открытой цели добавляем блок «Связи с другими целями» (после «Причин/Видения», перед «Планом»):

```text
СВЯЗИ

Над-цель
[ Здоровое тело            ✕ ]      ← если задана, чип с крестиком
[ + Выбрать над-цель          ]      ← если не задана

Под-цели (3)
• Пробежать полумарафон       ✕
• Тренировки 4x в неделю      ✕
[ + Добавить под-цель            ]
```

Оба действия («Выбрать над-цель», «Добавить под-цель») открывают один и тот же боттом-шит выбора цели:

- Список всех остальных целей с поиском.
- Скрыты: сама цель, её текущие потомки (для над-цели), её текущие предки (для под-цели), уже связанные.
- Внизу шита кнопка «+ Создать новую цель» — открывает обычный флоу создания и после сохранения сразу привязывает.

Удаление связи (крестик на чипе) — без подтверждения, действие обратимо в один тап.

## Технические детали

- Файл: `src/routes/_app.wishes.tsx` (модель, список, экран цели, новый шит).
- Хранение: там же, где сейчас лежат цели (тот же механизм persist, что и остальные поля `Goal`).
- Миграция: `parentGoalId` опциональный, для старых записей — `undefined`, работает без миграции.
- Утилиты в этом же файле:
  - `getChildren(goals, id)` — прямые под-цели.
  - `getDescendantIds(goals, id)` — все потомки (для защиты от циклов).
  - `getAncestorIds(goals, id)` — все предки.
  - `setParent(goalId, parentId | null)` — обновляет `parentGoalId`, отклоняет цикл.
- Компактная строка связей — небольшой inline-подкомпонент в файле карточки цели, стиль в духе существующих чипов раздела (оранжевый акцент, серый текст).

## Что НЕ делаем сейчас

- Не рисуем отдельный экран-дерево целей.
- Не поддерживаем несколько над-целей (граф) — только одна.
- Не трогаем задачи внутри целей и модуль ключевых задач.
